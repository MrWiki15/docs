## Enero — Infraestructura base

**Objetivo:** construir la base técnica y operativa de Kawi.

**Qué se logró:**

- Definición inicial del producto.
- Primeros flujos operativos.
- Estructura base de plataforma.
- Configuración inicial de servicios.
- Primeros procesos internos.

**Por qué fue importante:** Enero estableció la base para comenzar a procesar operaciones reales y validar el modelo.

## Febrero — Primeros 100 USD procesados

**Objetivo:** procesar las primeras operaciones reales.

**Qué se logró:**

- Validación inicial del flujo.
- Primeros usuarios usando la plataforma.
- Primer volumen transaccional.
- Confirmación de que el problema existía y los usuarios estaban dispuestos a usar Kawi.

**Cómo se consiguió:** canales de confianza, atención directa y soporte manual para acompañar las primeras operaciones.

## Marzo — Comunidad inicial

**Objetivo:** construir una comunidad inicial alrededor de Kawi.

**Qué se logró:**

- Primeros 100 miembros en comunidad.
- Primeros usuarios interesados recurrentes.
- Feedback directo de usuarios.
- Mayor confianza en el servicio.

**Cómo se consiguió:** principalmente mediante WhatsApp, recomendación orgánica y comunicación directa con usuarios.
