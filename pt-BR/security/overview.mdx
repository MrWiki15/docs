A segurança é uma prioridade para a Kawi.

## O que a Kawi deve proteger

- Dados de usuários.
- Operações.
- Saldos.
- API keys.
- Webhooks.
- Acesso a painéis.
- Histórico financeiro.

## Boas práticas

- Autenticação segura.
- Validação de operações.
- Monitoramento de atividade.
- Logs internos.
- Limites operacionais.
- Revisão de transações suspeitas.
- Separação entre sandbox e produção.

## Segurança para usuários

Os usuários devem receber:

- Confirmações claras.
- Histórico de operações.
- Suporte disponível.
- Status de transações.
- Transparência em taxas e comissões.

## Segurança para desenvolvedores

As empresas integradas devem:

- Proteger suas API keys.
- Validar webhooks.
- Usar HTTPS.
- Evitar expor credenciais.
- Implementar idempotência.
